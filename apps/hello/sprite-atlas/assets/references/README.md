# Reference PNGs -- image-reading protocol

> **Charter F2 (text over image) + F3 (AI self-report gate) + P5 (producer/consumer split).** Reference PNGs in this directory are dawn-node ground-truth baselines for pixel-parity comparison. They are binary artefacts, not text channels -- AI subagents MUST NOT inspect them visually.

## Protocol

| Step | Actor | Action |
|:--|:--|:--|
| 1 | ImplementerAgent | Commits the engine code and the smoke script (`scripts/smoke-dawn.mjs`) that generates the reference PNG |
| 2 | CI (dawn-node + lavapipe) | First run of `pnpm --filter @forgeax/hello-sprite-atlas smoke` writes `scripts/reference-dawn-walk-frame-0.png` and exits 1 (WRITTEN marker; force-add then rerun) |
| 3 | Orchestrator / Human | Inspects the written PNG visually, confirms correctness |
| 4 | All subsequent runs | Smoke compares frame output against `scripts/reference-dawn-walk-frame-0.png` with `eps <= 0.05` threshold |

## Filled-by field

Each reference PNG carries a metadata contract in the smoke script source (`scripts/smoke-dawn.mjs`):

```js
// reference.dawn.filledBy = 'orchestrator'
// Reference PNG is written on first CI run; subsequent runs compare against it.
// Subagents MUST NOT attempt to read or interpret the PNG visually.
// The orchestrator (human) confirms the baseline after visual inspection.
```

## Subagent self-report gate (charter F3)

When a subagent encounters a path pointing into this directory:

1. The subagent reads this README
2. The subagent reports: `"reference PNG at <path>: filledBy:'orchestrator' -- subagent cannot inspect; numeric delta-only from smoke script output"`
3. The subagent does NOT call `Read` on the `.png` file
4. If the smoke script emits a numeric delta, the subagent may report that delta as text
