# rhi-debug-viewer test fixtures

The engine repo tracks **no binaries** (zero-binary invariant, `grep:no-binary-assets`
gate). The hello-cube tape fixture is therefore **not committed** — it is synthesised
in memory at test time.

## Source of truth

- `build-hello-cube-tape.mjs` — exports `buildHelloCubeFixture()` returning
  `{ json, blob, report }` for a minimal valid tape (1 render pass + 1 draw + color
  attachment, 14 events). Built from the official `serializeTape` + `assembleReport` +
  `computePassOffsets` primitives, so the bytes match a real browser capture of the
  same event stream. The shader is a minimal fullscreen-triangle with a solid-color
  fragment; `wgslCode` is inline in the events, so the blob pool is empty.

## Consumers

- `src/__tests__/viewer-model.unit.test.ts` — calls `buildHelloCubeFixture()` then
  `deserializeTape(json, blob)` directly (no disk I/O).
- `scripts/smoke-browser.mjs` / `scripts/smoke-browser-no-webgpu.mjs` — build the
  fixture and write `frame-0.tape.bin` + `frame-0.report.json` into a throwaway
  `os.tmpdir()` directory for playwright `setInputFiles` (never committed).

## Local debugging

To inspect the generated bytes on disk (e.g. to load them in the viewer by hand):

```bash
node apps/rhi-debug-viewer/fixtures/generate-fixture.mjs [outDir]
```

Writes `frame-0.tape.bin` + `frame-0.report.json` into `outDir` (defaults to a fresh
`os.tmpdir()` directory). Do **not** write them into this `fixtures/` directory — the
`.bin` would be rejected by `grep:no-binary-assets`.

## Real browser capture (optional)

To capture a richer tape from a real WebGPU run instead of the synthetic minimal one:

```bash
FORGEAX_ENGINE_RHI_DEBUG=1 pnpm -F @forgeax/hello-cube smoke:browser
ls .forgeax-debug/<runId>/frame-0.*
```

`.forgeax-debug/` is gitignored — copy the bytes somewhere outside the repo, never
into `fixtures/`.

## Constraints

- Compute-mixed fixtures are NOT built here (D-6: hello-cube has no compute pass;
  compute fixtures are hand-built event arrays in the unit tests).
